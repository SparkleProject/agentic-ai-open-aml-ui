# Architectural Implementation Plan: FE-501 (Human Feedback RLHF Widget)

## 1. Executive Overview & Technical Strategy

**Task**: Build a Human Feedback (RLHF) Widget allowing analysts to rate AI agent output (👍/👎), correct errors, and annotate reasoning.

**Architectural Philosophy**:
Continuous improvement of LLM agents requires high-quality human-in-the-loop (HITL) feedback. By integrating a seamless Reinforcement Learning from Human Feedback (RLHF) widget directly into the analyst's workflow, we can collect granular data on hallucinations, misapplied policies, or missing context. This feedback dataset will be used in the future to fine-tune prompts and models.

## 3. Data & State Model

The component needs to manage the state of the feedback locally (until submitted).

```typescript
type FeedbackType = 'POSITIVE' | 'NEGATIVE' | null;

interface FeedbackPayload {
  caseId: string;
  feedbackType: FeedbackType;
  category?: 'HALLUCINATION' | 'MISSING_CONTEXT' | 'INCORRECT_POLICY' | 'OTHER';
  correctionText?: string;
}
```

---

## 4. Step-by-Step Implementation

### Step 1: Create `RLHFWidget.tsx`
- Build a React component utilizing `lucide-react` icons (`ThumbsUp`, `ThumbsDown`).
- **Initial State**: Show the two rating buttons inline.
- **Positive State**: If `ThumbsUp` is clicked, immediately show a "Thank you for the feedback" success state.
- **Negative State**: If `ThumbsDown` is clicked, expand the widget into a mini-form:
  - A `<select>` dropdown for the error category.
  - A `<textarea>` for the analyst to provide corrected reasoning.
  - A "Submit Feedback" `<button>`.
- **Submitted State**: After submission, replace the form with a success message indicating the feedback was logged for model tuning.

### Step 2: Integrate into `CaseWorkspace.tsx`
- Import `RLHFWidget`.
- Place the widget directly inside or immediately below the "Case Summary" `<Card>`, as this summary is generated by the AI agent and is the prime candidate for analyst evaluation.

### Step 3: UI/UX Styling
- Ensure the widget uses our established design system:
  - `hsl(var(--primary))` for active states.
  - `hsla(var(--bg-elevated))` for the expanded form background.
  - Smooth transitions when expanding the feedback form.

---

## 5. UI/UX Design System Mapping

| Concern | Approach |
|---|---|
| **Intrusiveness** | Keep the initial widget small (just two icon buttons). It should only expand if the user initiates a negative rating. |
| **Clear Categorization** | Provide predefined dropdown categories to make feedback easily quantifiable for future fine-tuning pipelines. |
